# دليل النشر على Railway 🚂

## الخطوات السريعة

### 1️⃣ التحضير

تأكد من أن جميع الملفات جاهزة:
- ✅ `requirements.txt` - قائمة المتطلبات
- ✅ `Procfile` - أوامر التشغيل
- ✅ `backend/main.py` - الخادم الرئيسي
- ✅ `public/` - ملفات الواجهة

### 2️⃣ رفع الكود إلى GitHub

```bash
# إضافة جميع الملفات
git add .
git commit -m "feat: Add Yemen news filter and new design2"
git push origin main
git push origin main --force
```

### 3️⃣ إنشاء مشروع على Railway

1. **افتح Railway:**
   - اذهب إلى https://railway.app
   - سجل دخول أو أنشئ حساب جديد

2. **إنشاء مشروع جديد:**
   - اضغط على "New Project"
   - اختر "Deploy from GitHub repo"
   - اختر مستودع `world-news`

3. **انتظر البناء:**
   - سيبدأ Railway تلقائياً في بناء المشروع
   - يستغرق عادة 2-5 دقائق

### 4️⃣ التحقق من النشر

1. **افتح السجلات (Logs):**
   - اضغط على تبويب "Deployments"
   - شاهد السجلات للتأكد من عدم وجود أخطاء

2. **احصل على الرابط:**
   - اضغط على "Settings"
   - اضغط على "Generate Domain"
   - انسخ الرابط

3. **اختبر التطبيق:**
   - افتح الرابط في المتصفح
   - تحقق من ظهور الواجهة بشكل صحيح
   - تحقق من تحميل الأخبار

## 🔧 إعدادات إضافية (اختيارية)

### تخصيص النطاق (Domain)

1. اذهب إلى Settings → Domains
2. اضغط على "Custom Domain"
3. أدخل النطاق الخاص بك
4. اتبع التعليمات لتحديث DNS

### ⚠️ إضافة Volume لقاعدة البيانات (مهم جداً!)

**بدون Volume، ستُفقد جميع البيانات عند كل إعادة نشر!**

1. **أضف Volume جديد:**
   - في Dashboard، اضغط على `+ New` → `Volume`
   - أو من الخدمة: `Settings` → `Volumes` → `+ Add Volume`

2. **اضبط الإعدادات:**
   - **Mount Path**: `/data`
   - اضغط `Create Volume` ثم `Attach`

3. **أعد نشر التطبيق:**
   - بعد إضافة Volume، أعد النشر
   - سيتم حفظ قاعدة البيانات في `/data/world_news.db`

### متغيرات البيئة (Environment Variables)

إذا كنت تريد إضافة متغيرات بيئة:

1. اذهب إلى Settings → Variables
2. أضف المتغيرات المطلوبة:
   ```
   PORT=8002
   PYTHON_VERSION=3.11
   ```

### زيادة الموارد

إذا كنت تحتاج موارد أكثر:
1. اذهب إلى Settings → Resources
2. اختر خطة أعلى (Hobby أو Pro)

## 📊 مراقبة التطبيق

### عرض السجلات (Logs)

```bash
# في Railway Dashboard
1. اضغط على المشروع
2. اختر "View Logs"
3. شاهد السجلات الحية
```

### مراقبة الأداء

- **CPU Usage**: تحقق من استخدام المعالج
- **Memory Usage**: تابع استخدام الذاكرة
- **Network**: راقب حركة البيانات

## 🐛 حل المشاكل الشائعة

### المشكلة: التطبيق لا يبدأ

**الحل:**
1. تحقق من `Procfile`:
   ```
   web: uvicorn backend.main:app --host 0.0.0.0 --port $PORT
   ```
2. تأكد من `requirements.txt` يحتوي على جميع المتطلبات
3. راجع السجلات للأخطاء

### المشكلة: الأخبار لا تظهر

**الحل:**
1. تحقق من السجلات لرسائل الخطأ
2. تأكد من أن yt-dlp يعمل بشكل صحيح
3. جرب إعادة تشغيل التطبيق

### المشكلة: قاعدة البيانات فارغة / البيانات تضيع عند إعادة النشر

**الحل:**
1. ✅ **تأكد من إضافة Volume** (انظر القسم أعلاه)
2. انتظر 3 دقائق (وقت الفحص الأول)
3. تحقق من السجلات - يجب أن تظهر:
   ```
   INFO:backend.main:Using database at: /data/world_news.db
   ```
4. إذا ظهر `./world_news.db` بدلاً من `/data/world_news.db`، فإن Volume غير مُضاف بشكل صحيح
